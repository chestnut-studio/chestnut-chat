# 07 - UI languages with `@nuxtjs/i18n`

## Goal

Add internationalization so the UI can switch between **English (`en`)** and
**Simplified Chinese (`zh`)**, persist the choice, and let the Settings modal
change the language live.

## Prerequisites

- `06-user-and-auth-ui.md` (the Settings modal has a language placeholder that
  this doc wires up).

## Context & files

Read first:

- `apps/web/nuxt.config.ts` - the Nuxt config; you add the module + options.
- `apps/web/app/components/SettingsModal.vue` - has the placeholder language
  control (marked `DOC 07`).
- Strings to translate live in the sidebar/settings/login/chat components built
  in docs 05, 06, 08, 10.

Files you will create / edit:

- **Edit** `apps/web/nuxt.config.ts`
- **Create** `apps/web/i18n/locales/en.json`
- **Create** `apps/web/i18n/locales/zh.json`
- **Edit** `apps/web/app/components/SettingsModal.vue`
- (Optionally) **Edit** components to replace hard-coded strings with `t('...')`.

## Background knowledge

- **`@nuxtjs/i18n`** is the official Nuxt i18n module. Add it to `modules` and
  configure under an `i18n` key.
- Locale files are JSON dictionaries. Reference keys in templates with the
  auto-imported `$t('key')`, and in `<script setup>` via
  `const { t } = useI18n()`.
- **Strategy**: use `strategy: "no_prefix"` so URLs do not change
  (`/dashboard` stays `/dashboard`). This is simplest for an app UI.
- **Persistence**: enable the browser-language detection + cookie so the choice
  survives reloads (`detectBrowserLanguage` with `useCookie: true`).
- **Switching at runtime**: `const { setLocale } = useI18n(); setLocale('zh')`.
  The current value is `locale` and the list is `locales`.
- **Nuxt 4 dir note**: with Nuxt 4 the i18n module expects locale files under an
  `i18n/` directory at the app root (i.e. `apps/web/i18n/locales/*.json`) and
  `langDir: "locales"`. Keep this exact layout to avoid path issues.

## Steps

### 1. Install the module

```bash
pnpm add @nuxtjs/i18n --filter web
```

### 2. Configure `apps/web/nuxt.config.ts`

Add `@nuxtjs/i18n` to `modules` and an `i18n` block. Keep the existing config;
only add the marked parts:

```ts
import "@chestnut-chat/env/web";

export default defineNuxtConfig({
  compatibilityDate: "latest",
  devtools: { enabled: true },
  experimental: {
    payloadExtraction: "client",
  },
  modules: ["@nuxt/ui", "@nuxtjs/i18n"],
  css: ["~/assets/css/main.css"],
  devServer: {
    port: 3011,
  },
  i18n: {
    strategy: "no_prefix",
    defaultLocale: "en",
    langDir: "locales",
    locales: [
      { code: "en", name: "English", file: "en.json" },
      { code: "zh", name: "简体中文", file: "zh.json" },
    ],
    detectBrowserLanguage: {
      useCookie: true,
      cookieKey: "i18n_locale",
      redirectOn: "root",
    },
  },
  runtimeConfig: {
    serverUrl: "",
    public: {
      serverUrl: process.env.NUXT_PUBLIC_SERVER_URL ?? "",
    },
  },
});
```

### 4. Wire the language control into the Settings modal

In `apps/web/app/components/SettingsModal.vue`, replace the placeholder language
options/binding (the block marked `DOC 07`) with i18n:

```ts
const { locale, locales, setLocale } = useI18n();

const languageOptions = computed(() =>
  (locales.value as { code: string; name: string }[]).map((l) => ({
    label: l.name,
    value: l.code,
  })),
);

const language = computed({
  get: () => locale.value,
  set: (value: string) => setLocale(value as "en" | "zh"),
});
```

The template's `<USelect v-model="language" :items="languageOptions" />` now
switches the locale live and persists it via the i18n cookie.

### 5. (Recommended) Replace hard-coded strings

Swap user-facing literals in the components from docs 05/06/08/10 for `$t`
keys, e.g.:

- `label="New chat"` -> `:label="$t('sidebar.newChat')"`
- group labels in `group-chats.ts`: keep the `key` (e.g. `today`) and render
  `$t('groups.' + group.key)` in the template instead of `group.label`.

This step is incremental; do the visible strings first.

- [ ] `@nuxtjs/i18n` is installed and added to `modules`.
- [ ] `en.json` and `zh.json` exist under `apps/web/i18n/locales/`.
- [ ] Settings modal language select lists English + 简体中文 and switches the
      UI language immediately.
- [ ] The chosen language persists across reloads (i18n cookie).
- [ ] At least the sidebar + settings + login strings use `$t(...)`.
- [ ] `pnpm run check-types` is clean.

```bash
pnpm run check-types
```

## Out of scope

- RTL languages, pluralization rules, and date/number localization beyond the
  basics.
- Translating long-form content (assistant messages stay as returned by the LLM).
